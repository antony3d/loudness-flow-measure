# Loudness Flow Measure (LFM) v0.8.2

Инструмент анализа громкости аудиофайлов по стандартам **ITU-R BS.1770-4** и **EBU 3342**. Выполняет расчёт интегральной громкости (LUFS), диапазона громкости (LRA) и многооконного анализа динамики (Loudness Flow). Строит визуальные отчёты в формате PNG.

[English](README.md) | **[Русский](README.ru.md)**

## Возможности

- **Integrated LUFS** — интегральная громкость по BS.1770-4 (двойное гейтирование)
- **LRA** — диапазон громкости по EBU 3342 (P10 / P95)
- **True Peak** — обнаружение межсэмпловых пиков через 2× оверсэмплинг
- **Digital Peak** — максимальный уровень сэмпла без оверсэмплинга
- **Loudness Flow** — анализ динамики на разных временных окнах (0.01–16 сек, логарифмический шаг)
- **Top-3 Dominants** — три наиболее встречаемых уровня громкости по длительности
- **Delta Comparison** — автоматическое сравнение треков с похожими именами
- **Overlay Flow** — при обработке ровно 2 файлов строится третий график с обеими кривыми Loudness Flow на одной оси для наглядного сравнения динамики
- **Визуализация** — графики:
  - `ds <имя>.png` — распределение плотности громкости
  - `fl <имя>.png` — график Loudness Flow
  - `fl <имя1>_vs_<имя2>.png` — overlay сравнение двух треков

## Требования

- Python 3.8+
- Библиотеки: `pydub`, `numpy`, `scipy`, `matplotlib`
- `ffmpeg.exe` и `ffprobe.exe` в папке `ffmpeg/`

## Установка

```bash
pip install pydub numpy scipy matplotlib
```

Убедитесь, что в папке проекта находятся:
```
ffmpeg/
├── ffmpeg.exe
└── ffprobe.exe
```

## Использование

```bash
python lfm.py <путь_к_файлу_или_папке>
```

Без аргументов обрабатывает все `.wav`, `.mp3`, `.flac` в текущей директории.

## Конфигурация (`lfm.ini`)

Файл `lfm.ini` содержит множество настроек вычислений и отображения:

- **[Main]** — режим (`LUFS`/`RMS`), verbose, опции `delta_comparison` (сравнение по имени) и `overlay_flow` (оверлей при 2 файлах)
- **[LoudnessDensity]** — диапазон оси X, количество столбцов, цвета гистограммы
- **[LoudnessFlow]** — окна анализа, отсечение пиков, масштаб оси Y, цвет линии

## Вывод

После обработки генерируются:
- `loudness_flow_report.txt` — текстовый отчёт
- `ds <имя>.png` — график плотности
- `fl <имя>.png` — график Loudness Flow
- `fl <имя1>_vs_<имя2>.png` — overlay сравнение (при 2 файлах, если включено)

## Алгоритм

1. **Загрузка** — декодирование через pydub, нормализация в float32
2. **K-фильтрация** — Pre-filter (High Shelf 1500 Гц) + RLB (High Pass 38.1 Гц)
3. **True Peak + Digital Peak** — 2× оверсэмплинг + поиск максимума на всех каналах
4. **Integrated LUFS** — расчёт momentary power (400 мс, шаг 100 мс), двойное гейтирование
5. **LRA** — расчёт по окнам 3 сек, определение P10/P95 с гейтированием
6. **Loudness Flow** — геометрическая серия окон, расчёт размаха (P95–P10)
7. **Визуализация** — сохранение PNG с настройками из `lfm.ini`

## Известные проблемы

- Поддержка только моно и стерео (>2 каналов не обрабатываются)
